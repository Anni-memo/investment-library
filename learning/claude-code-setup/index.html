00

この記事でできるようになること

この記事は、Claude Code を
「聞いたことはあるけれど、まだ触ったことがない人」
のための入口です。

難しいことはしません。
まずは次の状態まで連れていきます。

GOAL OF THIS ARTICLE
1. Claude Code をインストールできる
2. 起動できる
3. 最初の1回、会話できる
4. 最低限の CLAUDE.md を書ける

つまり、「使えるかもしれない」ではなく、「今日から使える」状態になることがこの記事の目的です。

Claude Codeは、Anthropic社が提供するAIコーディングアシスタントだ。ターミナル（コマンドライン）、デスクトップアプリ、Webアプリ、IDE拡張として動作する。

一般的なAIチャットとの違いは明確だ。Claude CodeはあなたのPC上のファイルを直接読み書きできる。コードを書くだけでなく、ファイルを整理し、Webを検索し、APIを叩き、テストを実行する。

プログラマーだけのツールではない。文章を書く人、データを整理する人、情報発信をする人—PCで何かを作るすべての人にとって、最強の相棒になる。

BEFORE — 普通のAIチャット
テキストを送り、テキストが返る。
それだけ。コピー&ペーストの往復。
AFTER — Claude Code
ファイルを読み、コードを書き、コマンドを実行し、結果を確認して修正する。
あなたの代わりに「手を動かす」AI。
01

まず結論 — 初心者の最短ルート

細かい説明は後にして、まず「何をすればいいか」だけ先に見せる。

FASTEST PATH — WINDOWS
1. Git for Windows をインストールする
2. PowerShell を開く
3. 下のコマンドを実行する
4. 新しいPowerShellを開いて claude と打つ

これだけで、最初のセットアップが始まる。
irm https://claude.ai/install.ps1 | iex

Macの場合はもっと簡単だ。ターミナルを開いて1行実行するだけ。

curl -fsSL https://claude.ai/install.sh | sh
ALSO OK
ターミナル操作が怖い方は、まずは Claude のデスクトップアプリ から始めても大丈夫。
ただし、ファイル操作や自動化まで深く使うなら、最終的には Claude Code（ターミナル版）を使えるようになるのがおすすめ。
02

Windowsで始める完全手順

Windowsは少しだけ準備が必要だ。でも手順通りやれば、10分で終わる。

IMPORTANT — PowerShell と CMD は別物
PS C:\Users\...> と出ていれば PowerShell。これを使う。
C:\Users\...> だけなら CMD（コマンドプロンプト）。別物なので注意。

PowerShell で CMD 用コマンドを打つと、&& エラーが出ることがある。
迷ったら、スタートメニューで「PowerShell」と検索して開こう。
Git for Windows をインストール
git-scm.com/downloads/win からダウンロード。インストーラーはすべてデフォルトのままで OK。
Git は Claude Code がファイル管理に使う。これがないと正常に動かない。
PowerShell を開く
スタートメニュー → 「PowerShell」で検索 → クリックで起動。
プロンプトが PS C:\Users\あなたの名前> になっていれば正解。
Claude Code をインストール
PowerShell に以下を貼り付けて Enter。
irm https://claude.ai/install.ps1 | iex
PowerShell を再起動する
インストール後、PowerShell をいったん閉じて、新しく開く。
これをしないと claude コマンドが認識されない。
Claude Code を起動
新しい PowerShell で以下を実行。
claude

初回起動時にブラウザが開き、ログイン画面が表示される。
Anthropic アカウント（Pro / Max / Team / Enterprise のいずれか）でログインすれば完了だ。

03

Macで始める完全手順

Mac はシンプルだ。ターミナルを開いて2つのコマンドを実行するだけ。

ターミナルを開く
Spotlight（⌘ + Space）→「ターミナル」と入力 → Enter。
または「アプリケーション → ユーティリティ → ターミナル」でもOK。
Claude Code をインストール
ターミナルに以下を貼り付けて Enter。
curl -fsSL https://claude.ai/install.sh | sh
Claude Code を起動
インストール完了後、そのまま以下を実行。
claude

Mac の場合、Xcode Command Line Tools のインストールを求められることがある。その場合は画面の指示に従ってインストールし、再度 claude を実行しよう。

04

Claude Code とブラウザ版・デスクトップ版の違い

Claude には複数の使い方がある。混乱しやすいので整理しておこう。

Claude Code（ターミナル / CLI） THIS ARTICLE
ターミナル上で動く。PC上のファイルを直接読み書きできる。コードを書き、コマンドを実行し、Git操作もする。最も強力。
デスクトップアプリ EASY START
claude.com/download からダウンロード。Claude Code と同等の機能をGUIで使える。ターミナルが苦手な人の入口として最適。
Webアプリ（claude.ai） NO INSTALL
ブラウザで claude.ai にアクセスするだけ。テキスト会話が中心。ファイル操作は限定的。
IDE拡張（VS Code / JetBrains） FOR DEVELOPERS
エディタ内でClaude Codeが動く。開発者向け。

プログラミング経験は不要。Claude Code自体がコードを書いてくれるので、あなたが書く必要はない。ただし「ターミナルを開く」「コマンドを入力する」という操作には慣れておきたい。

05

初回ログインで何が起こるか

claude を初めて実行すると、ブラウザが開いてログイン画面が表示される。Anthropic アカウントでログインすれば準備完了だ。

REQUIRED
Anthropicアカウント
claude.ai で登録。Pro または Max プラン、あるいは API（Console）で利用可能。
RECOMMENDED
まずは Pro / Max のいずれかで
使い放題の定額制で始められる。詳細は公式の価格ページを確認。
CAUTION — 課金の落とし穴
環境変数 ANTHROPIC_API_KEY が設定されていると、Proプランのサブスクリプションではなく API 従量課金が使われることがある。

サブスクで使いたい場合は、環境変数が設定されていないか確認しよう。
/login コマンドでアカウントの切り替えもできる。

ログインが成功すると、ターミナルに Claude Code の対話画面が表示される。ここからが本番だ。

06

最初に打つ5つのコマンド

Claude Code にログインしたら、まずこの5つを覚えよう。

1
claude
Claude Code を起動する。作業したいフォルダで実行するのがコツ。
2
/help
使えるコマンドの一覧を表示する。困ったらまずこれ。
3
/login
アカウントを切り替える。APIキーとサブスクを間違えたときにも使う。
4
/config
設定画面を開く。テーマやモデルの切り替えなど。
5
/memory
CLAUDE.md や自動メモリの状態を確認する。AIがあなたの何を覚えているか見える。
TRY IT
起動後、まず /help を打ってみよう。使えるコマンドが一覧で表示される。
次に「このフォルダの中身を教えて」と日本語で話しかけてみよう。Claude Code があなたの PC 上のファイルを読んで返答する。
07

CLAUDE.md の書き方

Claude Code をインストールしたら、最初にやるべきことがある。CLAUDE.md を書くことだ。

CLAUDE.md は、Claude Code が毎回起動時に読み込む「行動指針書」。ここに書いた内容が、AIの振る舞いを決める。

BEFORE — CLAUDE.md なし
初日の新入社員。
あなたのことを何も知らない。
毎回ゼロから説明が必要。
AFTER — CLAUDE.md あり
3年目の優秀なアシスタント。
あなたの好み・方針を知っている。
確認してから動く。

まずは1枚だけ。ホームディレクトリに ~/.claude/CLAUDE.md を作成しよう。最初から完璧を目指さなくていい。

1枚目: 全体共通の CLAUDE.md START HERE
ファイルの場所: ~/.claude/CLAUDE.md
すべてのプロジェクトで共通の設定。言語・口調・基本ルールを書く。
# 私の CLAUDE.md ## コミュニケーション - 日本語で応答してください - 簡潔に、要点を先に伝えてください - 保存や変更の前に、必ず確認してください ## 私について - 名前: （あなたの名前） - 職業: （あなたの職業） - 目標: （今取り組んでいること） ## 環境 - OS: Windows 11 - よく使うフォルダ: C:\Users\（ユーザー名）\Documents\

たったこれだけで、Claude Code の振る舞いは劇的に変わる。日本語で答え、簡潔に話し、勝手に変更せず確認してくる。

これは出発点に過ぎない。使いながら「こうしてほしい」「これはやめて」と気づいたことを追記していく。CLAUDE.md は生きたドキュメントだ。

2枚目: プロジェクト別の CLAUDE.md WHEN YOU'RE READY
ファイルの場所: プロジェクトフォルダ内の ./CLAUDE.md または ./.claude/CLAUDE.md
そのプロジェクト固有のルール。技術スタック・命名規則・特別な注意点を書く。
# プロジェクト CLAUDE.md（例） ## このプロジェクトについて - 名前: 自分のWebサイト - 技術: HTML + CSS（フレームワーク不使用） - 目的: 自己紹介ページを作る ## ルール - シンプルなHTMLを書く。複雑なフレームワークは使わない - 画像はすべて images/ フォルダに置く - コミットメッセージは日本語で
GROWTH PATH
最初は1枚で十分。慣れてきたら、プロジェクトごとに分ける。
Claude Code は ~/.claude/CLAUDE.md（全体共通）と ./CLAUDE.md（プロジェクト別）の両方を読む。
さらに 自動メモリ機能で、会話の中で学んだことも自動で蓄積されていく。

詳しくは → Part 2: 記憶を設計する
08

よくあるエラーと対処

初心者がハマりやすいポイントをまとめた。エラーが出ても慌てなくていい。ほとんどの場合、原因は単純だ。

irm is not recognized
CMD を開いている。
PowerShell を開き直そう。スタートメニューで「PowerShell」と検索。プロンプトが PS C:\... で始まっていれば正解。
claude is not recognized
PowerShell の再起動が必要。
インストール後、PowerShell をいったん閉じて新しく開く。それでもダメなら、PATH 環境変数を確認。
Git Bash が必要と言われる
Git for Windows が未導入。
git-scm.com からインストールしよう。
サブスクなのに API 料金が発生する
環境変数 ANTHROPIC_API_KEY が設定されている。
この変数があると、サブスクではなく API 課金が優先される。PowerShell で $env:ANTHROPIC_API_KEY を確認し、不要なら削除する。
Xcode Command Line Tools の
インストールを求められる（Mac）
画面の指示に従ってインストール。
数分かかるが、一度入れれば再要求されない。完了後に再度 claude を実行。
09

育てる — 使うほど賢くなる仕組み

Claude Code は使い捨てのツールではない。正しく設定すれば、使うほど賢くなる。

その仕組みが「メモリ」と「外部脳」だ。

メモリ（memory/） AUTOMATIC
Claude Code は会話の中で重要な情報を自動的にメモリに保存する。あなたの好み、プロジェクトの方針、過去のフィードバック。次回の会話でこれらを自動的に参照する。

詳しくは → Part 2: Claude Codeに記憶を設計する
外部脳との接続 ADVANCED
Obsidian などのノートアプリと連携させると、あなたの全知識にAIがアクセスできるようになる。知識への投資が複利で回り始める。

詳しくは → Part 3: 知識は複利で増える

最初の会話で試してほしいことを紹介しておこう。

試してみよう 1: ファイル整理 EASY
「ダウンロードフォルダの中身を種類別に整理して」
→ Claude Code がフォルダの中身を確認し、画像・文書・音楽などに分類してくれる。
試してみよう 2: 文章作成 PRACTICAL
「来週の会議用に、このプロジェクトの進捗報告を書いて」
→ プロジェクトフォルダの内容を読み、報告書を自動生成。
試してみよう 3: Webサイト作成 AMBITIOUS
「自己紹介のWebページを作って」
→ HTML/CSS を自動生成し、ブラウザで確認できる形で出力。このサイト自体、Claude Code で作っている。
最初の1時間は「設定」に使う。
その1時間の投資が、その後の100時間を変える。

11

さあ、始めよう

Claude Codeは、あなたが使うほどあなたのことを理解するAIだ。

最初は戸惑うかもしれない。「何を頼めばいいのかわからない」と。でも、それは新しいチームメンバーを迎えた初日と同じだ。最初はぎこちなくても、一緒に仕事を重ねるうちに、阿吽の呼吸になっていく。

今日インストールして、CLAUDE.md を1つ書く。それだけで、あなたとAIの関係は「使い捨て」から「育てる」に変わる。

この記事を読んだあなたは、もう始める準備ができている。

YOUR FIRST 30 MINUTES
0〜5分: Git をインストール（Windows の場合）
5〜10分: Claude Code をインストール → 起動 → ログイン
10〜20分: /help で探検 → CLAUDE.md を書く
20〜30分: 最初の会話を試す（ファイル整理がおすすめ）

30分後、あなたはもう「AIを育てる人」になっている。
